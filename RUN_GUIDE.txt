# Memory Allocation Visualizer - Quick Run Guide

This guide provides the sequence of commands required to build, set up, and run the complete Memory Allocation Visualizer stack on a Linux system.

---

## 1. Prerequisites
Ensure you have the following installed:
- gcc, make
- python3, python3-venv, pip
- node.js, npm

---

## 2. One-Time Setup

### A. Build the C Backend
```bash
cd backend
make
cd ..
```

### B. Setup the Python Bridge
```bash
cd bridge
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cd ..
```

### C. Setup the React Frontend
```bash
cd frontend
npm install
cd ..
```

---

## 3. Running the Project

You will need two separate terminal windows.

### Terminal 1: Python Bridge & Backend
The bridge will automatically launch the C backend binary.
```bash
cd bridge
source .venv/bin/activate
python3 bridge.py
```
*The bridge runs on http://localhost:5001*

### Terminal 2: React Frontend
```bash
cd frontend
npm run dev
```
*The UI runs on http://localhost:5000*

---

## 4. Common CLI Commands
Once the UI is running, you can enter these commands in the bottom command bar:

- `add P1 5`      : Add process P1 with 5 virtual pages.
- `kill P1`       : Terminate process P1.
- `algo lru`      : Switch to Least Recently Used algorithm (also 'fifo', 'opt').
- `frames 12`     : Change physical frame count to 12.
- `refs P1 1 2 3` : Set a custom page reference sequence for P1.
- `reset`         : Clear all processes and reset statistics.
- `status`        : Print memory state dump to the backend terminal.

---

## 5. Troubleshooting
- **Backend missing?** Ensure you ran `make` in the `backend/` directory.
- **Connection error?** Ensure the bridge is running in Terminal 1 before starting the frontend.
- **Stale State?** If the simulation hangs, press Ctrl+C in the bridge terminal and restart it.
